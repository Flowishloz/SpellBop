Regression test: `tests/test_collision_layers.gd`.
- **Dead spawn_offset_y export (Sprint 2 hotfix, 2026-06-10):** float @exports cached to fixed-point once in `_ready()` ignore runtime Inspector edits — symptom: "changing the export does nothing". FIX: `spawn_offset_y` now uses a setter that re-caches on write. RULE: any float @export whose fixed-point cache should respond to live tuning needs a re-caching setter. Also note: `scenes/test_area.tscn` overrides `spawn_offset_y=24.0` on its Player instance, shadowing `player.tscn` (10.0) and the script default (48.0) — per-instance scene overrides shadow tuning edits made elsewhere. Regression test: `tests/test_spawn_offset.gd`.
- **Origin spawn trap — local vs global fixed coordinates (Sprint 2 hotfix 2, 2026-06-10):** SpellCasterComponent read the caster's LOCAL `fixed_position` and wrote it into the projectile container's coordinate space — with the player positioned via a parent/wrapper or a container at a different origin, fireballs spawned at midfield (0,0). FIX: spawn path is global end-to-end via `get_global_fixed_position()` / `set_global_fixed_position()` (SG API has getter/setter METHODS, no property). RULE: never mix SG local fixed_position across different parents — cross-node placement must use the global fixed API. Regression test: `tests/test_spawn_offset.gd` TEST 4.
- **Frame-0 facing default (+Y) (Sprint 2 hotfix 2, 2026-06-10):** `cast_direction_y` defaulted to +1, but down-court toward the opposing wall is sim −Y — an unconfigured caster fired at its own baseline on frame 0. - **"Stuck at Y=0" — editor float transform never reaches the sim (Sprint 2 hotfix 3, 2026-06-11):** SG Physics 2D keeps the float Node2D transform and the fixed-point transform SEPARATE — dragging a node with the editor gizmo or setting float position/rotation writes ONLY the float side; the deterministic sim reads ONLY fixed_position/fixed_rotation. RULE: position/rotate SG nodes ONLY via the fixed_position_x/y / fixed_rotation Inspector fields (raw 64.16: sim units × 65536) — never the 2D viewport gizmo.
- **Physics rotation lock (Sprint 2 hotfix 3, 2026-06-11):** mandated guard — PlayerController._ready() hardcodes fixed_rotation = 0. Character facing is visual-only on the 3D rig. RULE: never rotate/mirror/negative-scale the SGCharacterBody2D or any SG ancestor for facing. - **Freed-instance == null re-capture (Sprint 3 tuning, 2026-06-11):** in Godot 4 a FREED object compares equal to null, so a test that guarded "capture once" with `if ref == null:` silently re-captured a fresh instance every time the old one was freed — masking a working lifespan feature as a test failure. RULE: never use `== null` to mean "not yet assigned" on refs that may be freed; pair is_instance_valid() with an explicit captured flag.

- **Anchor presets before parenting (Sprint 6, 2026-06-12):** calling set_anchors_preset() on a Control BEFORE add_child(), then setting position/size, recomputed offsets on tree entry and shoved a full-width banner off-center (caught by the screenshot probe). - **Sim-tick stage countdown vs wall-clock window desync (Sprint 7, 2026-06-12):** the staged spell's release was counted in SIM ticks while the on-screen countdown ran on the wall-clock Stack window — two clocks for one event drifted (playtest: "the spell releases as soon as the card appears"). FIX: the window closing IS the release — MatchController resolves the stack LIFO from TheStack.stack_closed; the sim-tick countdown was deleted. - **Container tick-order capture miss (Sprint 11, 2026-06-12):** sibling entities in the Projectiles container tick in CHILD ORDER, so whether the barrier scans before or after a ball moves depends on spawn order — a fast ball could bounce and travel a full tick-step out of the barrier's capture band between scans ("the WOA anticipation sometimes doesn't fire"). FIX: PlayerController polls press edges at RENDER rate and latches them into the next tick's input dict. - **Bare autoload identifiers in standalone compilation (Sprint 12, 2026-06-12):** scripts referencing the SoundFX autoload by name compiled in-game but FAILED --check-only/standalone loads ("Identifier not found") — the same console-noise class as the earlier editor spam. - **Freed player in a typed assignment — SFX crossfade crash (Sprint 13 hotfix, 2026-06-12):** SoundFX cached the last AudioStreamPlayer per sound; a naturally FINISHED player frees itself while the cache still holds the dead reference, and pulling that into a TYPED variable (`var prev: AudioStreamPlayer = dict.get(...)`) is a hard script error ("Trying to assign invalid previously freed instance") — crashing on the next retrigger of the same sound. - **First-draw pipeline compile hitch (Sprint 14, 2026-06-12):** scenes/materials instantiated mid-fight (the stack resolution spawning projectiles, BurstFX building a fresh StandardMaterial3D per burst) compile shaders/render pipelines on FIRST DRAW — a visible lag spike exactly at the slow-mo exit. RULE: anything that can be instantiated mid-fight must be drawn once during a loading beat; never construct fresh materials inside gameplay-time spawns.
- **Label autowrap property-order trap (Sprint 16, 2026-06-12):** setting a Label's `size` BEFORE enabling `autowrap_mode` snaps the control to its single-line minimum width, so the wrap never engages — descriptions rendered as one clipped line (found only by cropping/upscaling a screenshot). RULE: for code-built wrapped labels, set autowrap/clipping/font FIRST, size second, text last; and verify text layout in PIXELS, not in code review. Also logged: a stored-and-killed pattern for transient UI tweens (a stale ROUND-N fade ate the VICTORY title).
- **No Android binary for the native sim extension (Sprint 17, 2026-06-13):** sg-physics-2d — the GDExtension the ENTIRE deterministic sim depends on (SGFixed, SGCharacterBody2D, etc.) — ships ONLY windows/linux x86_64 binaries upstream; there is NO Android build for ANY release version (confirmed via the gitlab releases API). - **delta_rollback was dead weight, not a dependency (Sprint 17, 2026-06-13):** the delta_rollback GDExtension was loaded at startup but its classes (SyncManager etc.) are referenced ONLY in code COMMENTS as a future plan — never instantiated; the game runs on PlayerController's own local tick driver. FIX: disabled it by renaming addons/delta_rollback/bin/delta_rollback.gdextension -> .gdextension.disabled (and its .uid). - **Projectiles/walls spawned outside the arena (Sprint 17 hotfix, 2026-06-13):** every spawn site (_resolve_attack/_resolve_counter/_resolve_defense in card_caster_component.gd, _spawn_projectile in spell_caster_component.gd, split_on_barrier in fireball_controller.gd) placed the object at caster_pos.x with NO bounds clamp, so a WIDE object (Icey Retort frost wave, ±200u) spawned while the caster sat at the lane edge reached ~150u past the side walls. Regression: tests/test_spawn_bounds.gd. FORWARD-LOOKING CAVEAT (from the adversarial review): the bound is currently a SHARED global property (both wizards = 400, symmetric court), so the shard reading "the first wizard's mover" and the casters' from_float(400) defensive fallback are correct today. - **Wizard sprite sank into the floor — animator zeroed the sprite offset (Sprint 18 follow-up, 2026-06-14):** the charge-shake in wizard_animator_component.gd wrote `_sprite.position = Vector3(jitter)` while charging and reset it to `Vector3.ZERO` otherwise, EVERY frame — overwriting the Sprite3D's authored local offset (0, 1.13, 0) and dropping the sprite centre to y=0, so its lower half rendered below the floor plane. - **Staggered slow-mo resolution invisible at deep dilation (Sprint 20, 2026-06-14):** the stack's LIFO one-at-a-time release was mechanically correct (ignore_time_scale wall-clock timers, real gaps) but looked SIMULTANEOUS in play — at the 10% reaction dilation a 0.2 s real gap is only ~0.02 s of SIM time, so consecutive spells barely separated before normal speed resumed. - **GDScript lambda captures locals BY VALUE (Sprint 20 test, 2026-06-14):** a headless test captured a result into a local `Array` and a connected signal lambda did `arr = [true, ...]` to record the event — but GDScript lambdas capture local variables by VALUE, so REASSIGNING the captured variable inside the lambda never escaped, and the outer array stayed unchanged (the assertion failed even though the signal fired).
